# Mobile App Compatibility Guide: `warrantyDurationYears`

هذا الدليل مخصص لمطور تطبيق الموبايل حتى يجعل التطبيق متوافقاً مع التحديث الجديد لحقل الضمان.

الحقل الجديد المعتمد في الـ API هو:

- `warrantyDurationYears` (رقم صحيح، بالسنوات)

---

## الهدف

إظهار مدة الضمان في تطبيق الموبايل، والتعامل معها بشكل آمن في جميع الشاشات التي تعرض المنتج (قوائم + تفاصيل).

---

## 1) تحديث موديل المنتج في الموبايل

أضف الحقل التالي في Product Model / DTO:

- الاسم: `warrantyDurationYears`
- النوع: `int` أو `num` (يفضل `int`)
- القيمة الافتراضية: `0`

قاعدة مهمة عند parsing:

- إذا الحقل غير موجود أو `null` -> استخدم `0`
- إذا وصل رقم سالب لأي سبب -> حوّله إلى `0`

---

## 2) أين سيظهر الحقل في الـ API

الحقل أصبح جزءاً من استجابات المنتجات العامة والإدارية.

عملياً، ستجده في:

- endpoints قوائم المنتجات
- endpoint تفاصيل المنتج
- endpoint تفاصيل المنتج بالـ slug (إن كان التطبيق يستخدمه)

اسم الحقل في JSON دائماً:

- `warrantyDurationYears`

---

## 3) مثال استجابة يجب أن يدعمها التطبيق

```json
{
  "success": true,
  "data": {
    "_id": "65f0c7e2a19d0e0012ab34cd",
    "name": "Solar Inverter X",
    "warrantyDurationYears": 2,
    "pricing": {
      "finalPriceUSD": 1100
    }
  }
}
```

وفي حالة منتج قديم:

```json
{
  "warrantyDurationYears": 0
}
```

---

## 4) التعديل المطلوب في واجهات الموبايل

1. شاشة تفاصيل المنتج:
   - اعرض الضمان بصيغة واضحة مثل: `2 سنة` أو `0 سنة`.

2. بطاقة المنتج (Product Card) - اختياري:
   - يمكن عرض شارة صغيرة: `ضمان 2 سنة` إذا القيمة > 0.

3. الفلاتر (اختياري لاحقاً):
   - إذا احتجتم فلتر ضمان مستقبلاً، الحقل صار مناسب لأنه رقمي.

---

## 5) أفضل ممارسة للـ UX

- إذا القيمة `0`:
  - إمّا تعرض `بدون ضمان` أو `0 سنة` حسب سياسة المنتج.
- لا تعتمد على نصوص حرة من API؛ الحقل أصبح رقمي فقط.

---

## 6) اختبار التوافق (Checklist)

- [ ] Parsing لا يفشل إذا الحقل غير موجود.
- [ ] Parsing لا يفشل إذا الحقل `null`.
- [ ] يظهر الضمان في شاشة تفاصيل المنتج.
- [ ] لا يحدث crash في القوائم القديمة.
- [ ] تم اختبار منتج بقيمة > 0 ومنتج بقيمة 0.

---

## 7) ملاحظة Backfill للبيانات القديمة

تم تجهيز سكربت في الباكند لتعبئة المنتجات القديمة بالقيمة `0`:

- `backend/scripts/backfill-product-warranty-years.ts`

هذا يعني مستقبلاً كل المنتجات القديمة ستصبح موحدة على نفس الحقل.

---

## خلاصة سريعة للمطور

- أضف `warrantyDurationYears` في موديل المنتج بالموبايل.
- اجعل القيمة الافتراضية `0`.
- اعرضها في شاشة التفاصيل.
- تعامل معها كرقم سنوي فقط.
